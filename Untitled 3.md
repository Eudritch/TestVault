## Sync engine state machine (so UI stays consistent)

### Top-level states

- `idle_up_to_date`
    
- `idle_has_local_changes`
    
- `idle_has_remote_changes`
    
- `syncing`
    
- `offline`
    
- `error_auth`
    
- `error_network`
    
- `conflicts_detected`
    
- `resolving_conflicts`
    

### Key transitions

- Any → `offline` when network lost (don’t block editing)
    
- `syncing` → `conflicts_detected` when conflict appears
    
- `conflicts_detected` → `resolving_conflicts` on Resolve
    
- `resolving_conflicts` → `syncing` on Complete merge
    
- `syncing` → `idle_*` on success
    
- `syncing` → `error_*` on failure
    

UI rule: **only one “primary CTA”** at a time:

- Up to date → “Sync now” (secondary)
    
- Has changes → “Sync now” (primary)
    
- Conflicts → “Resolve” (primary)
    
- Error → “Fix / Reconnect” (primary)
